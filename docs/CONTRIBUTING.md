# Contributing to KindPath Q

## Principles
- Practical > clever.
- Mono-safe always.
- Explain decisions in plain English.
- Ship small, verify often.

## Branching
- main = stable
- feature/* = work branches

## Commits
Use conventional commits:
- feat: ...
- fix: ...
- chore: ...
- docs: ...
- refactor: ...

## Review Checklist
- Builds on macOS 12 Intel
- No obvious mono issues in analysis logic
- UI readable and minimal
- Doc updates included
